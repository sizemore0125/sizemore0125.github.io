Generics

We often write redundant code to perform similar operations on different types of data.

For example:

public void printArray(Integer[] array) {
    for (Integer element : array) { 
        System.out.print(element + " "); 
    } 
} 

public void printArray(Double[] array) { 
    for (Double element : array) { 
        System.out.print(element + " "); 
    } 
} 

public void printArray(String[] array) { 
    for (String element : array) {
        System.out.print(element + " ");
    }
}

Generics allow Java programmers to write flexible code without compromising the benefits of compile-time type checking. Generics are parameterized types, which allow classes, interfaces, and methods to operate on specific types defined at compile time.

Motivation for Generics

Suppose we implement a List abstract data type like this:

class List { 
    private int[] items = new int[10];
    private int length = 0;
    
    public int get(int index) {
        if (index >= length) {
            throw new IndexOutOfBoundsException(
                "Index " + index + " is out of bounds!"
            );
        }
        return items[index];
    }
    
    public void add(int item) {
        items[length] = item;
        length++;
    }
}

This is a poor implementation of the List abstract data type. While this List is dynamic in size, it can only contain ten elements. However, we will proceed with this implementation for now.

We can use our List class like this:

List list = new List();
list.add(1);
list.add(2);
list.add(3);
System.out.println(list.get(1));

This produces the following output:

2

This List implementation works for now, but it only supports values of type int. What if we want a List that can store values of type float?

One approach is to duplicate the code and replace all int types with float. However, this leads to code duplication and makes it difficult to maintain if we need to add new functionality to all List objects.

Another solution is to use the Object class instead of int, allowing us to store any type in our List:

class List { 
    private Object[] items = new Object[10];
    private int length = 0;
    
    public Object get(int index) {
        if (index >= length) {
            throw new IndexOutOfBoundsException(
                "Index " + index + " is out of bounds!"
            );
        }
        return items[index];
    }
    
    public void add(Object item) {
        items[length] = item;
        length++;
    }
}

Now, we can store any type of object in our List:

List list = new List();
list.add(1);
list.add(2.0);
list.add(new List());
System.out.println(list.get(1));

While this seems useful, it introduces problems. If we want to assign a value stored in this list to a variable, we must explicitly cast it, meaning we need to know the exact type of every stored value:

int i = (int) list.get(0);

Introducing Generics

Just as methods can have parameters, classes can also have parameters. Generics allow us to define a type parameter that represents the type of elements a class should handle. Using generics, we can define a type parameter T to represent the stored value type.

Here is a revised List class using generics:

@SuppressWarnings("unchecked")
class List<T> { 
    private T[] items = (T[]) new Object[10];
    private int length = 0;
    
    public T get(int index) {
        if (index >= length) {
            throw new IndexOutOfBoundsException(
                "Index " + index + " is out of bounds!"
            );
        }
        return items[index];
    }
    
    public void add(T item) {
        items[length] = item;
        length++;
    }
}

Note: We use @SuppressWarnings("unchecked") because generics do not work well with arrays in Java.

Example: Generic Container Class

Suppose we want to create a simple container class that holds a value of any type:

class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

The Box<T> class can store objects of any type, with type safety enforced at compile time. The <T> syntax allows Java to infer the correct type without explicit casting.

We can use the class like this:

Box<String> stringBox = new Box<>();
stringBox.set("Logan");
System.out.println(stringBox.get());

Box<Integer> intBox = new Box<>();
intBox.set(100);
System.out.println(intBox.get());

Generic Methods

Methods can also be made generic, allowing them to work with different types while maintaining type safety:

class Utility {
    public static <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.print(element + " ");
        }
        System.out.println();
    }
}

The printArray method takes an array of any type and prints its elements. The <T> syntax before the return type indicates that this is a generic method.

Example usage:


Integer[] intArray = {1, 2, 3, 4, 5};
String[] strArray = {"A", "B", "C"};

Utility.printArray(intArray);
Utility.printArray(strArray);

Bounded Type Parameters

Sometimes, we need to restrict the types that can be used with generics. For example, if we want to work only with numeric types, we can use bounded type parameters:

class NumberBox<T extends Number> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

Here, T extends Number ensures that T must be a subclass of Number, preventing inappropriate types such as String.

Example usage:

// NumberBox<String> stringBox = new NumberBox<>(); // This will cause an error

NumberBox<Integer> intBox = new NumberBox<>();
intBox.set(100);
System.out.println(intBox.get());

ArrayLists in Java

An ArrayList is a resizable array implementation of the List interface in Java, found in the java.util package. Unlike arrays, which have a fixed size, an ArrayList can dynamically grow and shrink as elements are added or removed.

To use an ArrayList, import java.util.ArrayList and specify the type of elements it will store:

import java.util.ArrayList;

ArrayList<String> names = new ArrayList<>();
        
names.add("Logan");
names.add("Ash");
names.add("Grant");
System.out.println(names);

Elements in an ArrayList can be accessed using the get method:

System.out.println(names.get(1));

Elements can be modified using the set method:

names.set(1, "David");
System.out.println(names);

Elements can be removed by index or by specifying the object:

names.remove(0);
names.remove("Grant");
System.out.println(names);

The number of elements in an ArrayList can be retrieved using the size method:

System.out.println(names.size()); // Output: 1

An ArrayList can be iterated using a for-each loop:

for (String name : names) {
    System.out.println(name);
}
