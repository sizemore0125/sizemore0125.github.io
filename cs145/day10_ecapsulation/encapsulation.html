Encapsulation

Let's say that we want to build a utility that can convert between different distances. Perhaps we could code something like this:

class Distance { 
	static final double METERS_PER_MILE = 1609.34; 
	static final double METERS_PER_KILOMETER = 1000; 
	
	static double convertMetersToKilometers(double meters) { 
		return meters / METERS_PER_KILOMETER; 
	} 
	
	static double convertMetersToMiles(double meters) { 
		return meters / METERS_PER_MILE; 
	} 
	
	static double convertKilometersToMeters(double kilometers) {
		return kilometers * METERS_PER_KILOMETER; 
	} 
	
	static double convertMilesToMeters(double miles) { 
		return miles * METERS_PER_MILE; 
	} 
}

Then, we could use this utility like this:

public class Main {
	public static void main(String[] args) { 
		double meters = 5000; 
		double kilometers = Distance.convertMetersToKilometers(meters); 
		double miles = Distance.convertMetersToMiles(meters);
		
		System.out.println("Distance in meters: " + meters);
		System.out.println("Distance in kilometers: " + kilometers);
		System.out.println("Distance in miles: " + miles); 
	} 
}

Consider the problems with this code:

  1. Conversion logic is spread across multiple functions without a cohesive relationship.
  2. Users of this utility must understand which functions to call and manage variables for each unit manually.
  3. Users might call conversion functions inconsistently, leading to calculation errors or redundant conversions.
    If this is "bad code", then how do we fix it?

In the following classes we are going to go into more depth about the guiding principles we should hold ourselves to when writing code in an Object Oriented Programming (OOP) language. We want our code to be understandable, easily maintainable, and extendable. So, how do we achieve that?

Abstraction

In OOP, Abstraction is a principle that focuses on only the essential details by hiding the complexity from the user. It allows us to simplify complex systems by breaking them down into smaller, more manageable parts.

For instance, when you use a microwave, you don't need to understand the inner workings of how it heats your food. You only interact with the buttons and functions provided to you. The complex details are abstracted away. This is what we want to achieve in our library management system: a simple interface that hides the complexity.

Encapsulation

In OOP, Encapsulation is a concept that wraps (or encapsulates) the data and the methods that work on the data within one unit. This prevents accidental modification of data. Encapsulation is the idea that a class can package some data together with the methods that manipulate the data.

Let's change our class to package up the data and the methods that operate on that data together. It wont have all the functionality that we want, but it will give us a starting point. This Distance class takes a parameters meters of type double, converts meters into two other units, and stores all three units as class attributes:

class Distance {
    double meters;
    double kilometers;
    double miles;

    static final double METERS_PER_MILE = 1609.34;
    static final double METERS_PER_KILOMETER = 1000;
    
    Distance(double meters) {
        this.meters = meters;
        this.kilometers = meters / METERS_PER_KILOMETER;
        this.miles = meters / METERS_PER_MILE;
    }
}

Access Modifiers

Our Distance class may be useful in certain circumstances, but it is also problematic. For example, we could easily reassign the attributes outside the class, meaning that the three distances are no longer equivalent:

Distance distance = new Distance(10.0);
distance.meters = 20.0;

System.out.println(distance.meters + " Meters);
System.out.println(distance.kilometers + "Kilometers");

Which will display:

20.0 Meters
0.01 Kilometers

These two distances are no longer the same, and so the usefulness of the class diminishes. However, we can fix this issue using access modifiers.

Access modifiers in Java control the visibility and accessibility of classes, methods, and variables. There are two that we will primarily use in this class:

  1. Public: attribute or method is accessible from any other class in any package.
  2. Private: attribute or method accessible only within the same class.

For example:

class Distance {
    private double meters;
    private double kilometers;
    private double miles;

    private static final double METERS_PER_MILE = 1609.34;
    private static final double METERS_PER_KILOMETER = 1000;
    
    public Distance(double meters) {
        this.meters = meters;
        this.kilometers = meters / METERS_PER_KILOMETER;
        this.miles = meters / METERS_PER_MILE;
    }
}

Now, if a anyone using the Distance class tries to access the meters attribute with dot notation, then it would result in this error:

error: meters has private access in Distance

By using the private access modifier, we prevent any erroneous modification in our Distance class. Typically, it is best practice to always have your attributes be private.

However, we now have a problem: there is no way to see what the meters, kilometers, and miles from our class. So, how do we address this? In this case, we use getter and setter methods to access these private attributes.

Getters/Setters

Getters and setters are methods used to access and modify private attributes of a class. They provide controlled access to these fields while maintaining encapsulation. With getters and setters, you control what data you want accessible, and how you want that data to be accessed.

Let's modify our Distance class with getters and setters:


class Distance {
    private double meters;
    private static final double METERS_PER_MILE = 1609.34;
    private static final double METERS_PER_KILOMETER = 1000;
    
    public Distance(double meters) {
        this.meters = meters;
    }
    
    public double getMeters() {
        return meters;
    }
    
    public double getMiles() {
        return meters / METERS_PER_MILE;
    }
    
    public double getKilometers() {
        return meters / METERS_PER_KILOMETER;
    }

    public void setDistanceInMeters(double meters) {
        this.meters = meters;
    }
    
    public void setDistanceInMiles(double miles) {
        this.meters = miles * METERS_PER_MILE;
    }
    
	public void setDistanceInKilometers(double kilometers) {
		this.meters = kilometers * METERS_PER_KILOMETER;
	}
}

In this updated class, we have implemented getters and setters for the three distance units supported. The getters allow users to retrieve the value of the distance in any of the three units. The setters update the meters attribute accordingly, ensuring consistency, regardless of the unit used for input.

Distance Class Refinement

We can further refine our Distance class. Perhaps a user wants to instantiate a Distance class, but they do not have their unit initially in meters. Perhaps our user has their distance measurement in miles or kilometers. In this case, we want our user to be able to create a Distance object regardless of the unit of the distance. 

class Distance {
    private double meters;
    private static final double METERS_PER_MILE = 1609.34;
    private static final double METERS_PER_KILOMETER = 1000;
    
    private Distance(double meters) {
        this.meters = meters;
    }

	// methods to create a Distance object.
    public static Distance fromMeters(double meters) {
        return new Distance(meters);
    }

    public static Distance fromKilometers(double kilometers) {
        return new Distance(kilometers * METERS_PER_KILOMETER);
    }

    public static Distance fromMiles(double miles) {
        return new Distance(miles * METERS_PER_MILE);
    }
    
    public double getMeters() {
        return meters;
    }
    
    public double getMiles() {
        return meters / METERS_PER_MILE;
    }
    
    public double getKilometers() {
        return meters / METERS_PER_KILOMETER;
    }

    public void setDistanceInMeters(double meters) {
        this.meters = meters;
    }
    
    public void setDistanceInMiles(double miles) {
        this.meters = miles * METERS_PER_MILE;
    }

	public void setDistanceInKilometers(double kilometers) {
		this.meters = kilometers * METERS_PER_KILOMETER;
	}
}
