Classes and Object-Oriented Programming

The best way to develop and maintain a large program is to construct it from small, simple pieces, or modules. Classes and methods help you modularize a program by separating its tasks into self-contained units. Use existing classes and methods as building blocks to create new programs.

Java requires that all code be written in a class. Thus far, we have generally been ignoring the functionality of a class. Now, we will begin to understand the power that classes give us when writing code.

We will learn about the fundamental concepts of OOP such as classes, objects, attributes, methods, and constructors. We'll also put this theory into practice by building a simple system to manage books in a library.

Object-Oriented Programming

Object-Oriented Programming (OOP) is a programming paradigm that relies on the concept of classes and objects. It is used to structure a program into simple, reusable pieces of code blueprints, which are used to create individual instances of objects.

Often an example of a car is used to introduce the concept of classes and objects. A car possesses certain properties, or attributes, like:

  • make
  • model
  • top speed
  • current speed
  • year of production
  • fuel capacity
  • etc.
    A car also has a number of actions that is can do, like:
  • turn on
  • drive
  • stop
  • refuel
  • break
  • change gear
  • etc.
    All of which might depend on the car's attributes.

Object oriented programming tries to package code into different classes of things. Then from these classes, we can create various different objects.

Key Concepts of OOP

  • Classes: These are the blueprint or prototype from which objects are created.
  • Objects: These are instances of classes.
  • Attributes: These are the data stored inside a class or instance and represent the state or quality of the class/instance.
  • Methods: In a class, methods are functions that define the behaviors of the class.
  • Constructor: A special kind of method that Java calls when it instantiates an object using the definitions found in your class.

We will look at all these concepts in detail in a moment.

Creating Classes

A class is a code template for creating objects. Objects have member variables and have behavior associated with them. In Java, a class is created by the keyword class:

class MyClass {
	// class members/methods
}

A quick note on notation, classes follow a Pascal Case naming scheme. Before, we were using Camel Case to name our variables.

Each class should have a single, well-defined responsibility, and the name of the class should express that responsibility.

From this class, we can then create an object, an instance of this class:

MyClass myObject = new MyClass();

The variable, myObject is a reference type, meaning that we must use the new keyword to instantiate this variable.

Attributes

In Java, a class can have a special set of variables called attributes. Attributes are variables that hold data about the class or the instance of the class (object).

For example:

class Book {
	String title = "Moby-Dick";
	int year = 1851;
}

Here we have created a Book class with two attributes: title and year. These variables will be associated with any instance of the class once it is instantiated and can be accessed using dot notation.

For example:

Book mobyDick = new Book();
System.out.println(mobyDick.title)
System.out.println(mobyDick.year)

This will display:

Moby-Dick
1851

Default Fields

When an attribute is declared but left uninitialized, they are automatically assigned default values based on their data type. These default values are determined by the Java language specification and ensure that fields always have a value.

Data Type Default Value
Numeric Types byte → 0
short → 0
int → 0
long → 0L
float → 0.0f
double → 0.0d
char '\u0000' (null character)
boolean false
Reference Types (e.g., objects, arrays) null

Methods

Perhaps we don't want all instances of our Book class to be Moby-Dick. We want our Book class to be able to represent any book. Let's use methods to do this.

In Java, a method is a block of code that performs a specific task. Methods are used to define the behavior of objects, reduce code duplication, and improve program organization.

Here is the syntax of a method:

returnType methodName(parameters) {     
	// method body 
}

There are four key components of a method in Java:

  • returnType: The data type of the value the method returns (e.g., int, String).
    • Use void if the method does not return a value.
  • methodName: A descriptive name following camelCase conventions.
  • parameters: Zero or more Inputs the method takes.
  • method body: The code block containing statements to be executed.

For example:

class Book {
	String title;
	int year;

	void setTitle(String bookTitle) {
		title = bookTitle;
	}

	void setYear(int bookYear) {
		year = bookYear;
	}
}

In this example, the attributes are left uninitialized initially. Two methods are then defined to set the attributes with the appropriate name and publication year. Note, the return type of these two methods are both void because we are not interested in returning any values for these two methods.

Let's use this class:

Book mobyDick = new Book();
mobyDick.setTitle("Moby-Dick");
mobyDick.setYear(1851);

Book walden = new Book();
mobyDick.setTitle("Walden");
mobyDick.setYear(1854);

System.out.println(mobyDick.name);
System.out.println(mobyDick.year);

System.out.println(walden.name);
System.out.println(walden.year);

This will display:

Moby-Dick
1851
Walden
1854

Constructors

It may be a bit cumbersome to manually set all the attributes of an instance of our Book class. Instead, it would make more sense if we set those values when we initially create the class. This is where constructors come in.

Constructors are special methods that are called when the new operator is used to create an object. Constructors can have parameters and do not have a return type. Classes can have multiple constructors through overloading. They are primarily used to initialize an object’s state at the time of creation.

Here is the syntax of a constructor:

class MyClass {
	// Class members
	
	MyClass() {
		// Initialization code
	}
	
	// Class methods
}

Note: Because we did not define a constructor for our Book class, a default constructor was automatically created by the compiler.

So, this code:

class Book {
	String title;
	int year;
}

Is equivalent to this code:

class Book {
	String title;
	int year;

	Book() { // empty constructor }
}

Let's make a constructor for our book class that allows us to set the title and year attribute when we instantiate a Book object:

class Book {
	String title;
	int year;

	Book(String bookTitle, int bookYear) {
		title = bookTitle;
		year = bookYear;
	}
}

Then we can create Book object like this:

Book mobyDick = new Book("Moby-Dick", 1851);

It is common practice for the parameter names in a constructor to match the names of the attributes being initialized. However, this can create ambiguity, as it becomes unclear which variable refers to the attribute and which refers to the parameter. To resolve this, the this keyword is used to explicitly refer to the object's attribute.

For example, we can update our Book class as follows:

class Book {
	String title;
	int year;

	Book(String title, int year) {
		this.title = title;
		this.year = year;
	}
}

Our Book class is looking good so far. Let's add an additional method displayInfo that neatly displays the book information: 

class Book {
	String title;
	int year;

	Book(String title, int year) {
		this.title = title;
		this.year = year;
	}

	void displayInfo() {
		System.out.println("Title: " + title);
		System.out.println("Publication Year: " + year);
	}
}

Now if we were to instantiate a Book object like this, and call our new method:

Book mobyDick = new Book("Moby-Dick", 1851);
mobyDick.displayInfo();

It would display this:

Title: Moby-Dick
Publication Year: 1851

Static Fields

If we want to build a library management system, perhaps we would want to keep track of the total number of books in the library. To do this we can give our Book class a static field.

A static field is an attribute that belongs to the class rather than a specific instance of the class. This means all instances of the class share the same static field, and changes to it are reflected across all instances.

I can create a static field using the static keyword. For example:

class Book {
	String title;
	int year;
	static int totalBooks = 0;

	Book(String title, int year) {
		this.title = title;
		this.year = year;
		totalBooks++;
	}

	void displayInfo() {
		System.out.println("Title: " + title);
		System.out.println("Publication Year: " + year);
	}
}

Now, every time a new the totalBooks field will increment by one. Meaning, that if we executed this code:

Book mobyDick = new Book("Moby-Dick", 1851);
Book walden = new Book("Walden", 1854);
System.out.println(Book.totalBooks);

This will display:

2

Notice that we are accessing a field that belongs to the class itself (e.g., Book.totalBooks).

Static Methods

Perhaps we want to create a method that compares the title two books. We could do something like this:

class Book {
	String title;
	int year;
	static int totalBooks = 0;

	Book(String title, int year) {
		this.title = title;
		this.year = year;
		totalBooks++;
	}

	void displayInfo() {
		System.out.println("Title: " + title);
		System.out.println("Publication Year: " + year);
	}

	boolean compareTitle(Book book) {
		return title.equals(book.title);
	}
}

And when we execute:

Book mobyDick = new Book("Moby-Dick", 1851);
Book walden = new Book("Walden", 1854);
System.out.println(mobyDick.compareTitle(walden));

This will display:

false

However, we could also implement this functionality as a static method. Static methods belong to the class rather than an instance of the class. This means they can be called without creating an object of the class. Static methods are often used for utility or helper functions that do not depend on object state.

*Note: It is bad practice to make all methods static!

Let's make compareTitle a static method using the static keyword:

class Book {
	String title;
	int year;
	static int totalBooks = 0;

	Book(String title, int year) {
		this.title = title;
		this.year = year;
		totalBooks++;
	}

	void displayInfo() {
		System.out.println("Title: " + title);
		System.out.println("Publication Year: " + year);
	}

	static boolean compareTitle(Book book1, Book book2) {
		return book1.title.equals(book2.title);
	}
}

Now, when we execute:

Book mobyDick = new Book("Moby-Dick", 1851);
Book walden = new Book("Walden", 1854);
System.out.println(Book.compareTitle(mobyDick, walden));

This will display:

false
